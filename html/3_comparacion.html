Comparación entre SQL y NoSQL

SQL (Relacional)

  • Estructura basada en tablas.
  • Esquema rígido y predefinido.
  • Uso de claves primarias y foráneas.
  • Relaciones entre entidades.
  • Lenguaje estándar SQL.
  • Alta integridad de datos.
  • Escalado vertical.

NoSQL (No relacional)

  • Basado en documentos, grafos o clave-valor.
  • Esquema flexible o inexistente.
  • No depende de relaciones fijas.
  • Datos anidados y dinámicos.
  • Consultas según motor de base de datos.
  • Escalado horizontal.
  • Optimizado para Big Data.
